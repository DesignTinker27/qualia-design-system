Spacing System ✅ Stable

Our 8px grid-based spacing system ensures harmonious layouts and consistent visual rhythm. Use semantic tokens for better readability.

Base Unit: 8px

All spacing adheres to an 8px base unit. This simplifies calculations and ensures consistent visual rhythm across the UI.

--qds-space-4: 16px; /* 8px × 2 */

Semantic Names

Prioritize semantic spacing tokens (`space.md`) over direct numeric values (`spacing[4]`) for enhanced readability and maintainability.

--qds-space-md: 16px;

Scale: 4px - 128px

A comprehensive, yet constrained, scale from micro (`4px`) to large layout gaps (`128px`) ensures flexibility without chaos.

--qds-space-xs: 4px;\n--qds-space-3xl: 48px;

For detailed spacing scale and common patterns, refer to the full tokens usage guide.

Usage Guidelines ✅ Stable

Our usage guidelines ensure that both designers and developers apply the Qualia Design System consistently, leading to a cohesive user experience and efficient workflows. These principles cover tokens, components, accessibility, and responsive design.

🎨 Design Tokens Best Practices

Always prioritize design tokens over hardcoded values. Use semantic names for clarity and follow the established scales to ensure consistency. Regularly test for accessibility and maintain a unified approach.

  • Always use tokens instead of hardcoded values.
  • Semantic naming over specific values.
  • Consistent spacing with 8px grid system.
  • Accessible colors with WCAG AA compliance.
  • Read the full token usage guide

🔘 Button Usage & Principles

Buttons should clearly communicate actions and intent. Use primary buttons for main actions, secondary for supporting roles, outline for subtle interactions, and destructive for critical operations. Always consider accessibility and responsiveness.

  • Primary: Main actions, CTAs (max 1 per section).
  • Secondary: Supporting actions, cancel buttons.
  • Outline: Subtle actions, tertiary options.
  • Destructive: Dangerous actions, deletion.
  • Read the full button usage guide

♿ Accessibility Standards

Accessibility is not an afterthought; it's integral to our design system. We adhere to WCAG 2.1 AA guidelines, ensuring our components are usable by everyone, regardless of ability or assistive technology.

  • Keyboard navigation for all interactive elements.
  • Focus indicators are clearly visible.
  • Color contrast meets WCAG standards (4.5:1 minimum).
  • Screen reader compatible with proper ARIA attributes.
  • Reduced motion respects user preferences.

📱 Responsive Design Principles

Our components and layouts are designed with a mobile-first approach. They adapt gracefully to various screen sizes, ensuring an optimal user experience across desktops, tablets, and mobile devices.

  • Mobile-first approach to component design.
  • Touch targets minimum 44px for mobile interactions.
  • Flexible layouts adapting to screen size.
  • Content priority on smaller screens.

Installation & Usage ✅ Stable

Get started with the Qualia Design System quickly. Follow these steps to install the package and integrate tokens and components into your project.

📦 Install Package

Add the Qualia Design System package to your project dependencies using npm or yarn.

npm install @qualia/design-system

🎨 Import Design Tokens

Integrate design tokens into your JavaScript/React components or directly into your CSS using custom properties.

// JavaScript/React\nimport { colors, spacing, typography } from '@qualia/design-system/tokens';\n\n// CSS (e.g., in your main CSS file or component styles)\n@import '@qualia/design-system/tokens/tokens.css';\n\n.my-component {\n background-color: var(--qds-color-primary-500);\n padding: var(--qds-space-md);\n}

🔘 Use Components

Import and use React components just like any other React component, applying variants and props as needed.

import { Button } from '@qualia/design-system';\n\nfunction MyApp() {\n return (\n <Button \n variant="primary" \n size="large"\n onClick={handleClick}\n >\n Get Started\n </Button>\n );\n}
\n

🔄 Migration from Semantic UI

Seamlessly transition your existing Semantic UI components to the Qualia Design System with clear mapping and guidance.

// Before (Semantic UI example)\n<Button primary size="large" loading={loading}>\n Submit\n</Button>\n\n// After (Qualia Design System)\n<Button variant="primary" size="large" loading={loading}>\n Submit\n</Button>
