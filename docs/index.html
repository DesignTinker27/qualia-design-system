Spacing System

Base Unit: 8px

All spacing follows an 8px grid system for visual consistency and developer ease.

--qds-space-4: 16px; /* 8px × 2 */

Semantic Names

Use semantic spacing names for better maintainability and readability.

--qds-space-md: 16px;

Scale: 4px - 128px

Comprehensive scale from micro spacing to large layout gaps.

--qds-space-1: 4px;
--qds-space-32: 128px;

Usage Guidelines

🎨 Design Tokens

  • Always use tokens instead of hardcoded values
  • Semantic naming over specific values
  • Consistent spacing with 8px grid system
  • Accessible colors with WCAG AA compliance

🔘 Button Usage

  • Primary: Main actions, CTAs (max 1 per section)
  • Secondary: Supporting actions, cancel buttons
  • Outline: Subtle actions, tertiary options
  • Destructive: Dangerous actions, deletion

♿ Accessibility

  • Keyboard navigation for all interactive elements
  • Focus indicators clearly visible
  • Color contrast meeting WCAG standards
  • Screen reader compatible with ARIA labels

📱 Responsive Design

  • Mobile first approach to component design
  • Touch targets minimum 44px for mobile
  • Flexible layouts adapting to screen size
  • Content priority on smaller screens

Installation & Usage

📦 Install Package

npm install @qualia/design-system

🎨 Import Design Tokens

// JavaScript/React import { colors, spacing, typography } from '@qualia/design-system/tokens'; // CSS @import '@qualia/design-system/tokens/tokens.css';

🔘 Use Components

import { Button } from '@qualia/design-system'; function MyApp() { return ( <Button variant="primary" size="large" onClick={handleClick} > Get Started </Button> ); }

🔄 Migration from Semantic UI

// Before (Semantic UI) <Button primary size="large" loading={loading}> Submit </Button> // After (Qualia Design System) <Button variant="primary" size="large" loading={loading}> Submit </Button>
